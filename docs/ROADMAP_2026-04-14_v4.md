## Milestone 1.7 — Migrate remaining route files to RBAC markers (1 week)

`admin-audit.ts` is the template (migrated in M1.5.4). The other route files
all need the same treatment. Process per file:

1. Wrap every handler in `protect/protectAll/protectAny/authRoute/publicRoute`.
2. Add the file path to `MIGRATED_FILES` in `tests/security/rbac-marker-coverage.test.ts`.
3. Run `npx vitest run tests/security/rbac-marker-coverage.test.ts` — must pass.
4. Update STATUS to record the migration count.

Priority order (high to low):

| File | Routes | Priority | Why |
|---|---|---|---|
| `server/routes/admin-rbac.ts` | ~12 | High | Admin-only, RBAC management — meta-critical |
| `server/routes/sdk-api.ts` | ~5 | High | Public API surface |
| `server/services/feature-flags/routes.ts` | ~14 | High | Highest traffic admin path |
| `server/services/feature-flags/site-routes.ts` | ~5 | High | Site-scoped feature flag operations |
| `server/routes/admin-sites.ts` | ~6 | High | Admin-only, site management |
| `server/routes/admin-users.ts` | ~6 | High | Admin-only, user management |
| `server/routes/incidents.ts` | ~7 | Medium | Incident management endpoints |
| `server/routes/messaging.ts` | ~25 | Medium | Largest file |
| `server/routes/automation.ts` | ~5 | Medium | Workflow management |
| `server/routes/projects.ts` | ~6 | Medium | Project CRUD |
| `server/routes/assistant.ts` | ~3 | Low | Smallest, safest pilot for the remaining migration |
| `server/routes/intelligent-watcher-admin.ts` | ~5 | Low | Niche admin tool |

**Exit criteria:** All ~12 route files migrated. Lint coverage at 100%. At
that point the soft-warning becomes "every new route file MUST be migrated
before merge."

## Milestone 1.4 — Postgres row-level security (1 week, requires DBA review)

Still deferred. Notes for whoever picks it up:

- Top 3 multi-tenant tables to start: `siteFeatureOverrides`, `featureRolloutHistory`, plus one new tagged table when added.
- The schemas in `shared/schema.ts` already have `siteId` columns; the policies need to enforce `siteId = current_setting('app.current_site_id')` or similar.
- The application layer needs to set the session variable per-request via the existing tenant isolation middleware.
- Bypass-attempt tests (try to read other-site rows via raw SQL or unscoped Drizzle) must be added to `tests/security/`.
- Rollout via a single migration in `migrations/`, gated behind a feature flag for safety.
