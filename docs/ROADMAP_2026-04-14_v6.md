## Milestone 1.8 — Expand tenant-scoped table registry (0.5 day, NEW in v6)

Phase A/B/C only protect the two tables currently registered in
`server/lib/tenant-scoped-registry.ts`. A full audit of the schema for
every table with a `site_id` column should follow, and each new
multi-tenant table should receive the same three-layer protection:

1. Registered in `tenant-scoped-registry.ts` (Layer 2).
2. Covered by a new policies migration extending `0003` (Layer 3).
3. Tested in `tests/rls/rls-isolation.test.ts` (Phase C).

Candidate tables (from a quick grep of `shared/schema.ts` for
`site_id` columns):

- `userFeatureOverrides` (if the `site_id` column exists — check)
- Audit log site columns
- Any CMS content table with site scoping

Exit: all multi-tenant tables are in the registry, all have policies,
all have RLS tests.
