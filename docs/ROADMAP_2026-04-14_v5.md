### Follow-up (tracked as M1.7-seed)

Four files use forward-referenced permission keys that are not yet in
the DB:

- `incidents:{read,create,update}` — 11 handlers
- `watcher:{read,manage}` — 19 handlers
- `automation:create` — 1 handler (reads use `authRoute`)
- `projects:{read,create,update}` — 8 handlers

`rbacService.hasPermission` fails closed on unknown keys, so these
behave as admin-only today via `user.isAdmin` bypass. The follow-up
migration should:

1. Add the resource names to `shared/rbac-schema.ts` `ResourceType`
   (or document that forward-referenced strings are an accepted
   pattern).
2. Seed the permissions in the RBAC setup migration.
3. Grant the permissions to appropriate default roles.
4. Update the TODO comments in each file to reference the seed
   migration number.

This is mechanical and can be batched with M1.4 (RLS) if the DBA
review covers both.
