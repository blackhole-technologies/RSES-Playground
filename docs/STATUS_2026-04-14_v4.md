## What changed since v3

### TypeScript baseline: 175 → 0

The v3 cleanup pass eliminated 75% of the 691-error baseline by fixing
seven cascade root causes. The remaining 175 errors were isolated bugs
spread across ~25 files. v4 worked through them systematically:

**Big files (10+ errors each):**

| File | Errors | Fix pattern |
|---|---|---|
| `server/cms/storage.ts` | 10 | Drizzle `$inferInsert` casts on `.values()` and `.set()` to bridge Zod-derived `string \| null \| undefined` with Drizzle's `string \| undefined` |
| `client/src/hooks/use-collaboration.ts` | 10 | Extended client-side `WSMessageType` union with collaboration types + corrected `autoReconnect` → `autoConnect` typo |

**Medium files (6-8 errors each):**

| File | Errors | Fix pattern |
|---|---|---|
| `server/services/feature-flags/pg-storage.ts` | 8 | Added missing `reason` to `SiteFeatureOverride` schema, added `performedBy` to `RolloutEvent`, made `userId` optional, cast targetingRules through `unknown` |
| `server/services/ai-modules/classification-module.ts` | 8 | Removed duplicate trailing `export {}` block + cast manifest config through `unknown` |
| `server/services/ai-module-system.ts` | 7 | `globalUsage.costThisMonth` → `totalCostThisMonth` (field rename), import `AIFallbackNotification` |
| `server/cqrs-es/actor.ts` | 7 | Moved `SupervisionStrategy` enum out of `import type` into value import; cast `this` to `ActorInstance<unknown, unknown>` for parent supervisor calls |
| `server/cms/routes.ts` (residuals) | 7 | Made `id` optional on insert schemas (storage layer derives it); `revisionId` cast for content updates |
| `server/services/file-watcher-cms.ts` | 6 | `WatchOptions` no longer exported by chokidar (define locally); `Stats` from `node:fs` not `fs/promises`; async-wrap event handlers |
| `server/routes/admin-sites.ts` | 6 | `String()` coercion on every `req.params.X` destructure |
| `server/multisite/index.ts` | 6 | Added local imports for `DomainRouter` and `ProvisioningService` (re-exports don't bring symbols into local scope) |
| `server/cqrs-es/opentelemetry-setup.ts` | 6 | Replaced `typeof import("@opentelemetry/sdk-node")` with `any` (OTel packages intentionally not in package.json) |

**Small files (1-5 errors each):** ~22 files, each with localized fixes — wrong field names, missing imports, narrow type literals, NodeJS.Timer vs Timeout, async handler signatures, covariance casts. See git history for per-file diffs.

**Patterns I established:**
- **`as unknown as T` for cross-shape casts** (Drizzle types, zod inference, OAuth2Tokens ↔ Record).
- **Closure-capture for object-literal `this`** (event-store.ts subscriptions).
- **`String(req.params.X)` per-call** (faster than fighting Express's bad `ParamsDictionary` declaration).
- **Explicit re-export blocks** when `export *` produces ambiguous names from sibling files.
- **Missing-fields-on-types** were generally fixed by adding the field to the schema (matching the runtime contract) rather than removing the call site.

### CI typecheck is now a hard gate

`.github/workflows/ci.yml` `typecheck` job had `continue-on-error: true` while
the 175-error baseline was being worked down. As of v4 it's removed. New
type errors will block merge. **Do not put it back.**
