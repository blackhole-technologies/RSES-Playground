**Files modified:**

- `server/lib/tenant-scoped.ts` — removed the WeakMap + three helpers; imports them from `tenant-scoped-tables.ts`; re-exports the public functions. Zero behavior change.

## When to ask the user vs. proceed autonomously

Carries forward from v2 with one addition under "proceed autonomously":

**Proceed autonomously:**

- Running the verification commands.
- Writing new RBAC policies, marker migrations, or test assertions that extend existing patterns.
- Fixing type errors you accidentally introduce.
- Updating `MIGRATED_FILES` or `tenant-scoped-registry.ts` as part of a larger migration you're shipping.
- Converting any newly discovered `db.*` → tagged-table bypass to `scoped()` / `withDbSiteScope()` (established in v7).
- **Relying on the dev-query guard as the fail-fast mechanism when refactoring multi-tenant code paths** (NEW — the guard will throw in dev if you forget a conversion; do not need to manually re-audit each call site after changes).

**Ask first:**

- Anything that touches `package.json` dependencies.
- Applying 0005 or 0004 (or any future one-way migration).
- Adding a new external service integration.
- Committing or pushing.
- Deleting any file that isn't generated/temporary.
- Any change that would lower the security test count or introduce a typecheck error.
- Re-adding `@anthropic-ai/sdk`.
- Registering a new multi-tenant table (implies migration + test work; user should know which table and why).
- Starting M1.8 (or any of its sub-milestones M1.8a–e) without a user priority pick.
