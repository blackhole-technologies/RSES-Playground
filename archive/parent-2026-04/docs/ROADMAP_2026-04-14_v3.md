## Milestone 1.7 — Migrate the rest of the route files (1 week)

`admin-audit.ts` is the template. The other route files all need the same
treatment:

| File | Routes (approximate) | Priority |
|---|---|---|
| `server/routes/admin-rbac.ts` | ~12 | High (admin-only, RBAC management) |
| `server/routes/admin-sites.ts` | ~6 | High (admin-only, site management) |
| `server/routes/admin-users.ts` | ~6 | High (admin-only, user management) |
| `server/routes/projects.ts` | ~6 | Medium |
| `server/routes/automation.ts` | ~5 | Medium |
| `server/routes/messaging.ts` | ~25 | Medium (largest file) |
| `server/routes/incidents.ts` | ~7 | Medium |
| `server/routes/sdk-api.ts` | ~5 | High (public API surface) |
| `server/services/feature-flags/routes.ts` | ~14 | High |
| `server/services/feature-flags/site-routes.ts` | ~5 | High |

**Process for each file:**
1. Wrap every handler in `protect/protectAll/protectAny/authRoute/publicRoute`.
2. Add the file path to `MIGRATED_FILES` in `tests/security/rbac-marker-coverage.test.ts`.
3. Run `npx vitest run tests/security/rbac-marker-coverage.test.ts` — must pass.
4. Update STATUS to record the migration count.

**Exit criteria:** All ~12 route files migrated. Lint coverage at 100%.
At that point, the soft-warning becomes "every new route file MUST be
migrated before merge."
