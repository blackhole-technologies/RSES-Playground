# System Documentation

## Overview

The Auto-Link System creates multiple organizational views of AI-generated projects using symlinks. Projects in `by-ai/` are automatically linked to topic, type, and filetype directories without duplicating files.

## Scripts

### auto-link-all.sh
Master script that runs all linkers and displays summary.

```bash
./auto-link-all.sh [--quiet] [--no-files]
```

Options:
- `--quiet` - Only output if changes occurred
- `--no-files` - Skip file-level linking (faster)

## Security Features

### Source: `lib/security.zsh`

#### Boundary Validation
All paths are validated to stay within allowed boundaries:
```bash
ALLOWED_BOUNDARIES=("$HOME/Projects" "$HOME/Dev")
```

Symlinks cannot escape these directories. Attempts are logged and blocked.

#### Sensitive File Blocking
These patterns are never linked:
- `*.env`, `*.env.*`, `.env`
- `*secret*`, `*secrets*`
- `*.pem`, `*.key`, `*.p12`, `*.pfx`
- `*password*`, `*.crt`, `*.keystore`
- `id_rsa*`, `id_ed25519*`, `id_ecdsa*`
- `.netrc`, `.npmrc`, `.pypirc`
- `credentials*`, `token*`, `api_key*`

#### Sensitive Directory Protection
Never descend into:
- `.ssh`
- `.aws`
- `.gnupg`
- `.docker`
- `.kube`
- `.config/gcloud`

#### Path Traversal Prevention
Any path containing `..` is blocked and logged.

#### Input Sanitization
`sanitize_name()` removes:
- Path traversal attempts (`../`)
- Null bytes and control characters
- Leading/trailing dashes and dots
- Truncates to 255 characters

#### Rate Limiting
Maximum 5000 files per run to prevent resource exhaustion.

#### Audit Logging
All operations recorded to `~/.audit.log`:
```
[2026-01-29T10:30:00] [user] [SYMLINK_CREATED] target -> source
[2026-01-29T10:30:01] [user] [BLOCKED_SENSITIVE] .env blocked
```

## Skip Patterns

### Source: `lib/ignore.zsh`

#### Why Skip Patterns?
- **Dependencies** (node_modules) can contain 100,000+ files
- **Build outputs** are generated, not source
- **IDE files** are user-specific
- **VCS directories** (.git) are metadata
- **Cache files** are temporary

#### Hardcoded Patterns

**SKIP_DEPS (Dependencies):**
```
node_modules, bower_components, __pycache__, *.pyc, venv, .venv,
vendor, .bundle, target, .gradle, .m2, packages, .nuget
```

**SKIP_BUILD (Build Outputs):**
```
dist, build, out, output, bin, obj, lib, .next, .nuxt, .vercel,
.netlify, _site, public, .docusaurus
```

**SKIP_IDE (Editor Files):**
```
.vscode, .idea, .vs, *.swp, *.swo, *~, .project, .classpath,
*.sublime-project, *.sublime-workspace
```

**SKIP_VCS (Version Control):**
```
.git, .svn, .hg, .bzr, CVS
```

**SKIP_CACHE (Caches):**
```
.cache, .tmp, .temp, .pytest_cache, .mypy_cache, .eslintcache,
.turbo, .parcel-cache, coverage, .nyc_output
```

**SKIP_OS (OS Files):**
```
.DS_Store, .AppleDouble, ._*, Thumbs.db, Desktop.ini
```

#### Priority Order
1. **Per-project `.gitignore`** - Most specific, project-aware
2. **User `.projectignore`** - Your custom patterns
3. **Hardcoded patterns** - Sensible defaults

## .projectignore Usage

Location: `~/Projects/.projectignore`

### Syntax
```bash
# Comments start with #
# One pattern per line
# Supports glob patterns: * ? [abc] [!abc]

# Example patterns
*.min.js          # Minified JavaScript
test-results      # Test output directories
*.generated.ts    # Generated TypeScript
```

### Sections in Template
1. **Additional dependency directories** - Package managers not in defaults
2. **Additional build outputs** - Project-specific build dirs
3. **Additional IDE/editor files** - Less common editors
4. **Logs and runtime files** - Generated at runtime
5. **Test artifacts** - Test outputs and fixtures
6. **Large files & media** - Binary files, archives, video

### Adding Custom Patterns
Edit `~/.projectignore` and add patterns at the bottom:
```bash
# My custom patterns
my-generated-dir
*.custom.ext
specific-project-*/output
```

## Troubleshooting

### Scripts Hang on Large Projects
Ensure find exclusions include all dependency directories. Check if a project has unusually deep nesting.

### Symlinks Not Created
1. Check `~/.audit.log` for BLOCKED entries
2. Verify source path is within allowed boundaries
3. Check if file matches sensitive patterns

### Statistics Show High Skip Counts
This is normal - a single node_modules can contain thousands of directories. High skip counts indicate the ignore system is working.

### Changes Not Appearing
Run `auto-link-all.sh` without `--quiet` to see all operations. Check if the project matches prefix/suffix patterns.
